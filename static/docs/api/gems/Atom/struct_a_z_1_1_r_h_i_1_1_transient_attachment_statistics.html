enum AZ::RHI::TransientAttachmentStatistics::AllocationPolicy
strong
Enumerator
HeapPlacement 

This policy is used when the platform is using resource placement onto memory heaps. This may include aliasing of memory. In this case, resources will overlap on the heap at different scopes.

ObjectPooling 

This policy is used when the platform is using simple object pooling. In this case, the heap offsets should be ignored, and each heap instance is is treated as a pool of disjoint attachments. The user can sum the total.
