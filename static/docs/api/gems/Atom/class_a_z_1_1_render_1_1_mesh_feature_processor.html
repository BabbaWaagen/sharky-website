Data::Instance<RPI::ShaderResourceGroup> AZ::Render::MeshFeatureProcessor::GetObjectSrg ( const MeshHandle &  meshHandle) const
overridevirtual

Gets the ObjectSrg for a meshHandle. Updating the ObjectSrg should be followed by a call to QueueObjectSrgForCompile, instead of compiling the srg directly. This way, if the srg has already been queued for compile, it will not be queued twice in the same frame. The ObjectSrg should not be updated during Simulate, or it will create a race between updating the data and the call to Compile
